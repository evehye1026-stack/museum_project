# 국립중앙박물관 전시 코스 안내 서비스 — PRD

> 서비스명(가안): **처마**
> 디자인 레퍼런스: [`design.md`](./design.md)
> 데이터 레퍼런스: [`public/courses.csv`](./public/courses.csv) (박물관명 / 추천코스명 / 전체 관람 시간 / 전시관명 / 전시실명 / 진열장 번호 / 유물명, 250행)

---

## 1. 개요

국립중앙박물관 방문객이 **(1) 미리 짜인 전시 코스를 둘러보거나, (2) 자신의 취향에 맞는 코스를 추천받거나, (3) 유물을 도감 형태로 찾아보고, (4) 유물과 직접 대화하듯 질문**할 수 있는 웹 서비스.

기존 리포지토리에는 `courses.csv` 기반으로 박물관 선택 → 코스 목록을 보여주는 최소 구현(`App.jsx`, `MuseumSelector`, `CourseCard`)이 있음. 본 PRD는 이를 4개 핵심 기능(전시 코스 / 취향 코스 추천 / 유물 도감 / 유물 챗봇) 중심의 서비스로 확장하는 것을 목표로 한다.

### 배경
- 국립중앙박물관은 전시실이 넓고 유물 수가 많아, 방문객이 짧은 시간 내에 무엇을 봐야 할지 결정하기 어려움.
- 기존 오프라인 안내(리플렛, 도슨트)는 개인 취향을 반영하지 못하고, 유물에 대한 궁금증을 그 자리에서 해소하기 어려움.

### 목표
| 목표 | 성공 지표(예시) |
|---|---|
| 방문 전/중 코스 계획 시간을 단축 | 코스 선택까지 걸리는 클릭 수·시간 |
| 취향 기반 추천으로 만족도 향상 | 추천 코스 채택률, 완주율 |
| 유물에 대한 이해도·흥미 제고 | 유물 도감 조회 수, 챗봇 대화 턴 수 |

### 타깃 사용자
- 국립중앙박물관 방문 예정/방문 중인 일반 관람객(내외국인)
- 시간이 제한적이라 효율적인 동선이 필요한 사용자
- 유물에 대해 좀 더 깊이 알고 싶은 사용자(어린이 동반 가족, 학생, 역사 애호가 등)

### 범위 (In / Out of scope)
- **In**: 코스 열람, 취향 기반 코스 추천, 유물 도감 검색/상세, 유물 페르소나 챗봇
- **Out (이번 버전 제외)**: 실내 위치 기반 내비게이션(GPS/비콘), 티켓 예약·결제, 다국어(한국어 우선, 영어는 향후), 회원 소셜 기능

## 2. 정보 구조 (IA) / 상단 내비게이션

`design.md`의 헤더 컴포넌트(스티키, 흰 배경, 좌측 로고 + 우측 검색/프로필 아이콘) 구조는 유지하고, **중앙 메뉴 라벨을 아래로 확정**한다.

```
┌──────────────────────────────────────────────────────────┐
│ [로고:처마]   전시 코스   취향 코스 추천   유물 도감   🔍  👤  │  ← 상단바 (sticky header)
├──────────────────────────────────────────────────────────┤
│                                                            │
│   ✦ 취향 코스 추천 칸 (상단바 바로 아래, 랜딩 최상단 섹션)      │
│   "지금 이 순간, 당신에게 맞는 코스는?"                        │
│   [테마 선택 칩] [시간 선택 칩]  →  [추천 코스 카드]           │
│                                                            │
│   전시 코스 목록 / 추천 코스 리스트 (design.md 6.4 카드 그리드) │
│                                                            │
└──────────────────────────────────────────────────────────┘
                                          [💬 유물과 대화하기] ← 플로팅 챗봇 버튼(전 화면 공통)
```

- 상단바 메뉴 3개는 각각 별도 라우트/페이지로 연결된다: `/courses`(전시 코스), `/for-you`(취향 코스 추천), `/artifacts`(유물 도감).
- **취향 코스 추천 칸**은 메뉴 이동 없이도 즉시 쓸 수 있도록 **랜딩 페이지(홈) 최상단, 상단바 바로 밑**에 위젯 형태로 노출한다 (요청사항 반영).
- **유물 챗봇**은 특정 페이지에 종속되지 않고 전 화면에서 접근 가능한 **플로팅 버튼 + 슬라이드 패널**로 제공한다.
- 모바일에서는 `design.md` 5.2 구조를 따라 상단 메뉴 3개는 하단 탭바로 이동, 챗봇 버튼은 하단 탭바 위에 플로팅으로 겹치지 않게 배치.

## 3. 핵심 기능 명세

### 3.1 전시 코스 (`/courses`)
사전에 구성된 추천 코스(예: "한국의 보물")를 열람하는 기능. `design.md` 6.1의 **코스 카드 컴포넌트**를 그대로 사용.

- 코스 리스트: 코스명, 전체 관람 시간, 경유 전시관 수, 대표 유물 썸네일
- 코스 상세: 대표 코스 카드(네이비 그라데이션) + Step 타임라인(전시관 → 전시실 → 유물 순서로 구성) + 유물 목록
- 데이터 소스: `courses.csv`의 `추천코스명` 그룹 → 그룹 내 `전시관명/전시실명/유물명` 순서를 Step으로 변환

**Step 매핑 규칙(제안)**: 동일 `전시관명` 내 연속 행을 하나의 Step으로 묶고, Step의 소요 시간은 `전체 관람 시간`을 Step 개수로 균등 분배하거나(1차), 이후 전시관별 실측 가중치를 반영(2차 개선).

### 3.2 취향 코스 추천 (`/for-you` + 홈 위젯)
사용자가 간단한 취향 입력만으로 코스를 추천받는 기능. 홈 화면에서는 축약형 위젯으로, `/for-you`에서는 전체 플로우로 제공.

**입력 (택 1 또는 조합)**
- 관심 테마 칩: 예) 조각/불교미술, 도자공예, 금속공예·왕실, 회화·서화, 역사·고고
- 관람 가능 시간 칩: 30분 / 1시간 / 2시간 이상
- (선택) 동행 유형: 혼자 / 가족·아이 동반 / 연인·친구

**출력**
- 추천 코스 카드 1~3개 (매칭 스코어 or 랭킹 순), `design.md`의 코스 카드 스타일 재사용
- "다시 추천받기" 액션으로 재질문 가능

**추천 로직 (1차 — 규칙 기반)**
- `courses.csv`의 코스별 총 관람시간과 사용자의 시간 조건을 매칭
- 코스별 대표 전시관/테마 키워드를 태깅(수동 or 유물명 기반 간단 분류)하여 관심 테마와 매칭
- 2차 개선: 사용자 클릭/완주 데이터를 반영한 협업 필터링 또는 LLM 기반 추천으로 고도화

### 3.3 유물 도감 (`/artifacts`)
유물을 도감(encyclopedia) 형태로 탐색·검색하는 기능.

- 목록: 카드/그리드 뷰, 유물명, 소속 전시관·전시실, 대표 이미지, 소속 코스(있다면) 뱃지
- 검색/필터: 유물명 검색, 전시관·전시실 필터
- 상세 페이지: 유물명, 위치(전시관 > 전시실 > 진열장 번호), 이미지, 설명 텍스트, **"이 유물과 대화하기"** CTA(3.4 챗봇 진입점)
- 데이터 갭: 현재 `courses.csv`에는 유물 설명/이미지 컬럼이 없음 → 별도 유물 메타데이터(설명, 이미지 URL, 시대, 재질 등)가 필요 (섹션 6 참고)

### 3.4 유물 챗봇 — "유물과 대화하기"
사용자가 특정 유물을 **1인칭 페르소나**로 삼아 대화할 수 있는 챗봇.

- **진입점**: (a) 유물 도감 상세 페이지의 "이 유물과 대화하기" 버튼, (b) 전 화면 공통 플로팅 챗봇 버튼(유물 미선택 시 유물 선택 유도 또는 기본 안내 페르소나)
- **UI**: 우측/하단 슬라이드 패널 채팅 UI. 상단에 대화 중인 유물명 + 썸네일 고정 노출, 하단 입력창 + 추천 질문 칩(예: "너는 언제 만들어졌어?", "어디서 발견됐어?")
- **페르소나 규칙**: 유물의 시대·용도·발견 배경 등 메타데이터를 시스템 프롬프트에 반영해 1인칭으로 답변(예: "나는 백제 시대에 만들어진 금동대향로야...")
- **기술 방향**: LLM API(예: Claude) 호출 + 유물별 컨텍스트 주입. 사실 왜곡 방지를 위해 알려진 메타데이터 범위 내에서만 답하도록 가드레일 필요(모르는 정보는 "정확히는 알려지지 않았어" 식으로 답하도록 지시)
- **비속어/부적절 입력 처리**: 유물 페르소나 톤을 벗어나지 않는 선에서 정중히 주제를 되돌리는 응답

## 4. 화면 흐름 (User Flow)

```
[홈]
 ├─ 취향 코스 추천 위젯에서 테마/시간 선택
 │    └─ 추천 코스 카드 클릭 → [코스 상세]
 ├─ 상단바 "전시 코스" → [코스 목록] → 카드 클릭 → [코스 상세]
 │    └─ [코스 상세] 내 유물 클릭 → [유물 상세] → "대화하기" → [챗봇 패널]
 ├─ 상단바 "취향 코스 추천" → [추천 플로우 전체 화면] → [코스 상세]
 └─ 상단바 "유물 도감" → [유물 목록] → 검색/필터 → [유물 상세] → "대화하기" → [챗봇 패널]

[챗봇 패널]은 어느 화면에서도 플로팅 버튼으로 열 수 있으며, 페이지 이동과 무관하게 대화 컨텍스트(선택된 유물)를 유지한다.
```

## 5. 디자인 가이드 연계 (`design.md` 참조)

| 요소 | 적용 방식 |
|---|---|
| 컬러 팔레트 | 코스 카드·추천 위젯 강조 배경은 딥네이비 그라데이션(`#1B2A5E → #5C73A5`), CTA는 `#0078FF` |
| 타이포그래피 | 코스명·유물명은 명조(Noto Serif KR), 메뉴/버튼/챗봇 UI는 고딕(Noto Sans KR) |
| 헤더 | 좌측 로고 + 중앙 메뉴(전시 코스/취향 코스 추천/유물 도감) + 우측 검색·프로필 아이콘, sticky |
| 코스 카드 | `design.md` 6.1 그대로 — 전시 코스, 취향 추천 결과, 도감 내 "소속 코스" 뱃지에 공통 재사용 |
| Step 타임라인 | `design.md` 6.2 — 코스 상세 페이지의 전시관/전시실 이동 순서 표시에 사용 |
| 챗봇 패널(신규) | 기존 팔레트를 계승한 신규 컴포넌트 — 배경은 Surface White, 유물 발화 버블은 Deep Navy 텍스트, 사용자 버블은 Bright Blue 계열로 구분(디자인 시안 추가 필요, 8절 TODO) |
| 반응형 | `design.md` 8절 브레이크포인트(모바일 <768px) 그대로 적용, 챗봇은 모바일에서 풀스크린 모달로 전환 |

## 6. 데이터 모델 (제안)

### 6.1 기존 — `courses.csv`
`박물관명, 추천코스명, 전체 관람 시간, 전시관명, 전시실명, 진열장 번호, 유물명`

### 6.2 신규 필요 — 유물 메타데이터 (`artifacts` 테이블/JSON)
유물 도감·챗봇 페르소나에 필요. 현재 CSV에는 없으므로 추가 수집/입력 필요.

| 필드 | 설명 |
|---|---|
| 유물명 | courses.csv와 조인 키 |
| 시대 | 예: 백제, 고려, 조선 |
| 재질/기법 | 예: 금동, 청자 상감 |
| 설명 | 도감 상세·챗봇 시스템 프롬프트용 짧은 해설 |
| 이미지 URL | 도감/코스 카드 썸네일 |
| 발견/소장 배경 | 챗봇 답변용(선택) |
| 테마 태그 | 취향 코스 추천 매칭용(예: 불교미술, 왕실공예) |

### 6.3 신규 필요 — 코스 테마 태그 (취향 추천용)
`추천코스명` 별로 테마 태그·난이도·추천 동행 유형을 매핑하는 별도 룩업 테이블.

### 6.4 챗봇 세션(런타임, 저장 불필요할 수도 있음)
- 선택된 유물 ID, 대화 히스토리(턴별 role/content) — 세션 스토리지 또는 서버 저장 여부는 개인정보/로그 정책에 따라 결정 필요(8절 오픈 이슈)

## 7. 비기능 요구사항

- **성능**: 코스/도감 목록은 최초 로드 후 클라이언트 캐싱, 챗봇 응답은 스트리밍으로 체감 지연 최소화
- **접근성**: `design.md` 9절과 동일하게 네이비 배경 대비 텍스트 명도 WCAG AA 검증
- **반응형**: 데스크톱/모바일 모두 지원 (`design.md` 8절 기준)
- **콘텐츠 정확성**: 챗봇은 보유한 유물 메타데이터 범위를 벗어난 확정적 주장을 하지 않도록 가드레일 적용
